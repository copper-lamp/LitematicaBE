## 一、实现方法概述

### 1.1 核心技术方案

本插件是一款基于Minecraft Bedrock版服务端的投影辅助工具，功能原型为Java版Litematic模组。插件采用以下核心技术方案实现完整的投影功能：

**渲染方案：直接方块放置（Direct Block Placement）**

插件直接在世界中放置方块来显示投影内容。通过设置方块ID和数据值来呈现目标方块。这种方案实现简单直接，但会在世界中留下实际方块。通过切换显示/隐藏状态来管理投影的可见性。

**存储方案：JSON格式 + 二进制格式**

投影数据支持两种存储格式：
- JSON格式：兼容Java版Litematica导出的.litematic文件
- 二进制格式（开发中）：使用字典编码压缩，理论压缩比可达6:1至15:1

**交互方案：表单系统（SimpleForm）**

使用LeviLamina提供的表单API构建GUI菜单，包括主菜单、原理图浏览器、当前投影控制、设置、帮助界面。玩家通过游戏内表单进行所有操作，无需外部工具。

### 1.2 关键实现步骤

插件开发分为以下主要阶段：

第一阶段实现项目基础架构，包括目录结构搭建、manifest.json配置、基础命令注册。

第二阶段实现GUI菜单系统，包括主菜单界面、原理图浏览器、当前投影控制界面、设置界面、帮助界面。

第三阶段实现投影数据管理，包括JSON文件解析、二进制序列化（开发中）、文件扫描读取。

第四阶段实现投影渲染，包括方块放置与清除、显示/隐藏切换、位置移动。

### 1.3 技术选型依据

**选择Node.js而非Lua的原因**

Node.js是LeviLamina官方推荐的现代脚本引擎，具有以下优势：与前端开发技术栈一致，学习成本低；支持更丰富的JavaScript特性；生态系统丰富，便于引入第三方库；异步编程模型适合处理IO操作。

**选择直接方块放置的原因**

相比实体渲染方案，直接方块放置实现更简单，不需要处理实体生命周期管理。这种方案适合原型开发和快速迭代。后续可升级为实体渲染以支持更高级的功能。

## 三、系统架构说明

### 3.1 模块划分

插件采用模块化架构设计，目前实现的核心模块：

**主入口模块（index.js）**

主入口模块负责插件的初始化和生命周期管理。主要职责包括：插件启动时的初始化流程、命令注册和事件监听器绑定、全局状态管理、模块间协调调度。

**GUI模块（index.js内的表单函数）**

GUI模块负责所有用户界面的实现。包括：
- showMainMenu() - 主菜单表单
- showSchematicBrowser() - 原理图浏览器
- showCurrentProjection() - 当前投影控制
- showSettings() - 设置表单
- showHelp() - 帮助界面

**存储模块（index.js内的加载函数）**

存储模块负责投影数据的读取和管理。包括：
- loadSchematic() - 从文件加载原理图
- getSchematicList() - 扫描目录获取文件列表
- saveSchematicBinary() - 二进制保存（开发中）
- loadSchematicBinary() - 二进制加载（开发中）

**渲染模块（index.js内的渲染函数）**

渲染模块负责投影的可视化显示。包括：
- createProjection() - 创建投影对象
- removeProjection() - 移除投影
- placeProjectionBlocks() - 放置方块显示
- clearProjectionBlocks() - 清除显示方块
- toggleProjection() - 切换显示/隐藏
- moveProjection() - 移动投影位置

### 5.2 部署流程

**第一步：确认基础环境**

确保BDS服务端正常运行，LeviLamina框架已正确安装。执行命令确认LeviLamina加载成功，日志中应显示"LeviLamina loaded"信息。

**第二步：安装插件**

1. 将LitematicaBE插件文件夹复制到plugins目录
2. 确保文件夹名称为LitematicaBE
3. 内部包含manifest.json和index.js
4. 在schematics目录下放置原理图文件

**目录结构：**
```
bedrock-server-1.21.x/
├── bedrock_server_mod.exe     # 服务端主程序
├── plugins/
│   ├── LeviLamina/            # LeviLamina核心
│   │   └── ...
│   ├── legacy-script-engine-nodejs/  # Node.js引擎
│   └── LitematicaBE/          # 投影插件
│       ├── manifest.json
│       ├── index.js
│       └── schematics/         # 原理图存储目录
│           └── (投影文件)
```

**第三步：首次启动**

启动BDS服务，插件将自动加载。控制台显示以下信息表示插件启动成功：
```
[LitematicaBE] LitematicaBE v1.0.0 Loading...
[LitematicaBE] LitematicaBE v1.0.0 Loaded!
[LitematicaBE] Commands: /litematica, /lit
```

**第四步：验证安装**

玩家进入游戏，执行命令测试插件功能。输入 `/litematica` 或 `/lit` 打开主菜单。如菜单正常显示，表示安装成功。

### 5.3 日常维护

**定期备份**

建议每周备份一次投影文件目录。在进行重大操作（如大规模投影加载）前，应手动备份相关文件。备份可使用压缩工具，保留历史版本。

**日志监控**

定期检查插件日志，关注错误和警告信息。异常日志可能预示潜在问题。日志文件位置：bedrock-server-1.21.x/logs/latest.log

**性能监控**

监控服务端TPS变化，特别是在加载大型投影时。如TPS显著下降，考虑减少同时显示的投影数量。

**版本更新**

更新插件前务必备份配置和投影文件。更新后检查新版本是否需要迁移配置文件。更新后首次运行应测试所有核心功能。

### 5.4 注意事项

**文件权限**

确保BDS运行用户对插件目录和投影存储目录有读写权限。Windows系统下注意防病毒软件可能误报。

**投影大小限制**

单次加载的投影建议不超过10万方块。超大型投影可能导致加载缓慢或内存不足。可分区域多次加载进行建造。

**维度切换**

投影数据和放置方块与维度绑定。切换维度后，当前投影方块不会自动跟随。返回原维度后需要重新放置投影。

**玩家离线处理**

当玩家离线时，其投影会自动隐藏（通过清除方块实现）。再次上线时需要重新执行显示操作。

## 七、常见问题与解决方案

### 7.1 安装部署问题

**问题：插件加载失败**

表现：服务端启动日志显示错误，插件功能不可用。

排查步骤：
1. 检查LeviLamina是否正确安装并加载
2. 检查manifest.json格式是否正确
3. 查看详细错误日志，定位具体失败原因

解决方案：
- 确保LeviLamina版本兼容
- 检查JavaScript语法错误
- 检查文件路径是否正确

**问题：命令无法使用**

表现：输入命令后提示未知命令或无权限。

排查步骤：
1. 检查命令是否正确注册
2. 查看日志确认命令注册状态

解决方案：
- 确认命令前缀正确（/litematica 或 /lit）
- 检查日志中的注册信息

### 7.2 功能使用问题

**问题：投影加载后不显示**

表现：材料清单显示加载成功，但游戏内看不到投影。

排查步骤：
1. 检查是否执行了 `/litematica place` 命令
2. 检查是否在可视范围内
3. 检查维度是否正确

解决方案：
- 加载原理图后必须执行place命令放置投影
- 移动到投影所在位置
- 确认当前维度与放置维度一致

**问题：方块位置不正确**

表现：投影方块位置与预期不符。

排查步骤：
1. 检查放置位置的坐标
2. 使用 `/litematica move` 调整位置

解决方案：
- 投影以玩家位置下方一格为原点
- 使用move命令精细调整位置

### 7.3 性能问题

**问题：加载大型投影时服务器卡顿**

表现：加载时TPS显著下降，玩家体验变差。

排查步骤：
1. 检查投影大小（方块数量）
2. 监控实体数量

解决方案：
- 减少同时显示的方块数量
- 分批次加载大型投影
- 使用更小的投影文件

```json
{
    "Metadata": {

        ]
    },
    "Entities": []
}
```

**格式说明**
- pos: 方块相对位置 [x, y, z]
- id: 方块ID（数字）
- data: 方块数据值（0-15）
- nbt: NBT数据（可选）
